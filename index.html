Wie ich Web Entwickler wurde

Schon früh interessierte ich mich für Computer und kreative Software: Ich bearbeitete Bilder mit Photoshop, schnitt Videos mit Premiere, experimentierte mit VFX in After Effects und folgte ersten 3D-Tutorials in Blender. Ich probierte vieles aus, lernte ständig dazu und tauchte immer tiefer in die Welt der digitalen Gestaltung ein.

Vor etwa zwei Jahren entdeckte ich dann meine Begeisterung fürs Programmieren. Was als Hobby begann, wurde schnell zu einem konkreten Berufswunsch. Seitdem habe ich mir die Webentwicklung mit Kursen, Tutorials und viel Eigeninitiative selbst beigebracht.

Was mich am Programmieren so fasziniert, ist, dass ich mit einigen Zeilen Code Konzepte und Designs zum Leben erwecken kann – funktional und visuell. Besonders spannend finde ich, wie man durch kleine Details wie Animationen und interaktive Features eine Benutzeroberfläche lebendig gestalten kann.

Inzwischen habe ich eigene Webprojekte, Spiele und Apps realisiert – darunter Cosmio, eine interaktive 3D-Webseite über das Universum, sowie LiveKick, eine Fußball-App mit Live-Daten, die über einen selbst erstellten Express-Proxy-Server von einer API abgerufen werden.

Cosmio

Projektbeschreibung

Cosmio ist eine interaktive Webseite und Progressive Web App, die es Nutzern ermöglicht, das Universum in 3D zu erkunden und Informationen über Himmelsobjekte abzurufen. Für eine beeindruckende visuelle Erfahrung kommen der Google Model Viewer sowie eingebettete Modelle über Sketchfab zum Einsatz. Die 3D-Modelle sind im glTF-Format eingebettet und mithilfe der glTF-Pipeline komprimiert, um die Ladezeit zu optimieren. Ich habe dieses Projekt gewählt, um gezielt mit 3D-Objekten im Web zu experimentieren und mein Verständnis für deren Integration, Darstellung und Optimierung zu vertiefen.

LiveKick

Projektbeschreibung

LiveKick ist mein bisher umfangreichstes und technisch anspruchsvollstes Projekt – und zugleich mein erstes mit Backend-Anbindung. Es handelt sich um eine Fußball-Webseite und Progressive Web App, die über einen selbst entwickelten Express-Server Daten aus den APIs von football-data.org und TheSportsDB abruft und im Frontend dynamisch rendert. Angezeigt werden aktuelle Spielstände und Tabellen aus sechs großen Ligen: Serie A, Bundesliga, La Liga, Premier League, Ligue 1 und der Champions League. Ich habe dieses Projekt gewählt, um gezielt den Umgang mit API-Fetch-Requests sowie das dynamische Rendern externer Daten im Frontend zu erlernen.

Tech Stack

  • HTML
  • CSS
  • JavaScript
  • Git
  • GitHub
  • Node.js
  • Express
  • Render
  • PWA

Features

Express Proxy Server

Um Spiel- und Tabellendaten ohne CORS-Einschränkungen abrufen zu können und die Fetch-Limits der APIs einzuhalten, habe ich einen kleinen Express-Server entwickelt. Dieser ruft die Daten von den externen APIs ab, speichert sie im Cache und stellt sie über definierte GET-Endpunkte für das Frontend bereit. So wird sichergestellt, dass die Daten regelmäßig – einmal pro Minute – aktualisiert werden, ohne das Request-Limit der API's zu überschreiten. Beliebig viele Nutzer können diese Daten dann vom Server abrufen, ohne selbst Requests an die API zu senden.

Ligen

Nutzer können zwischen verschiedenen Ligen wechseln und jeweils die aktuellen Spiele sowie Tabellen abrufen.

Favoriten

Nutzer können Spiele zu den Favoriten hinzufügen, welche im Local Storage gespeichert bleiben.
